🌐/translate Admin

英⇄日 双方向自動翻訳。指定チャンネルで投稿された英文/日本文に対して、bot が自動で訳文をreplyします。

コマンド

使い方説明
/translate onこのチャンネルで自動翻訳を有効化
/translate offこのチャンネルの自動翻訳を停止
/translate mode direction:<both|en2ja|ja2en>翻訳方向を変更
/translate status有効チャンネル一覧、プロバイダ、月使用量を表示
/translate text text:<文字列> [direction:<...>]任意文字列を即時翻訳

使用例

john_smithcard-game · 10:24
Hello, my package hasn't arrived yet.
SupportBot✓ BOTcard-game · 10:24
🇯🇵 こんにちは、まだ荷物が届いていません。

翻訳方向 (mode)

値動作
both （既定）英→日、日→英 双方向
en2ja英→日のみ。日本人同士の会話チャンネル向け
ja2en日→英のみ。海外向け案内チャンネル向け
off停止（/translate offと同じ）

🤖/suggest Admin

全体交流会チャンネル（COMMUNITY_CHANNEL_ID）の発言をAIで分類し、商品問合せ／配送相談に該当する場合のみ誘導文をreply。

コマンド

使い方説明
/suggest on意図判定機能を有効化
/suggest off停止
/suggest status現在の状態を確認

使用例

takuyafree-talk · 14:10
こんばんは！PSA10のピカチュウ151ありますか？
SupportBot✓ BOTfree-talk · 14:10
@takuya 商品のお問い合わせは #ticket でチケットを開いてお聞きください🎫
※ AI自動判定です（信頼度 92%）。違ったらスルーしてください。

分類ラベル

  • product_inquiry — 商品の在庫/価格/状態/取置き → 誘導文を返す
  • shipping_inquiry — 配送/送料/追跡/関税 → /shipping案内＋誘導文を返す
  • general_chat — 雑談/挨拶 → スキップ
  • other — その他 → スキップ
  • 信頼度しきい値 SUGGEST_MIN_CONFIDENCE（既定0.70）未満は反応なし
  • 同一メッセージへの reply は1回限り（連投チェック）
  • 投稿者の言語に応じて日本語／英語を自動切替

📦/shipping 全員

カート式UIで複数商品＋宛先国を選択し、Google Sheets「比較表US基準」を参照して送料を算出。

コマンド

使い方説明権限
/shipping送料計算パネルを ephemeral で起動全員
/shippingadmin reload商品マスタとスプシキャッシュを再読込Admin

パネル操作フロー

  1. /shipping 実行 → ephemeralパネルが表示
  2. 🎁 商品 を選択（6種類）
  3. 🔢 数量 を選択（1, 2, 3, 5, 10, 20, 50, 100, その他）
  4. 🏳️ 宛先 を選択（リージョン → 国、または 🔍検索）
  5. 3つ全部入力後、緑の 📥 送信（カートに追加） でカートへ
  6. 同じ操作で複数商品を追加可能。宛先はリセットされない
  7. 🧮 送料を計算 ボタンで結果表示
  8. 必要なら 📤 公開する でチャンネルにシェア

商品マスタ

商品絵文字重量(g/個)単位
わんぴーすCASE🧰5,300個
わんぴーすBOX🍞400個
ポケモンCASE🧰4,000個
ポケモンBOX⚡300個
PSA🏆40枚
Single card🃏3枚

🎉/giveaway Admin

時間指定の抽選ツール。参加者は🎉ボタンを押すだけ。期限切れで自動的に当選者発表。

コマンド

使い方説明

/giveaway end message_id:<ID>早期終了して即座に当選者発表
/giveaway reroll message_id:<ID> [winners]終了済み抽選の再抽選

期間 (duration) フォーマット

入力意味
30s30秒
5m5分
1h1時間
1d1日

使用例

/giveaway create prize:Amazonギフト1000円 duration:1h winners:3
/giveaway create prize:限定カード duration:1d winners:1 image:[添付] required_role:@VIP

⚙翻訳の仕組み

プロバイダ切替

.env の TRANSLATE_PROVIDER で選択。

値用途コスト必要キー
deepl （推奨）日常翻訳Free 100万字/月DEEPL_API_KEY
claude意訳寄り、文脈理解従量課金ANTHROPIC_API_KEY

処理フロー

  1. on_message イベントを受信
  2. bot/コマンド/プレフィックス済(🇯🇵🇺🇸)はスキップ
  3. 長さチェック（2文字以上 / 2000文字以下）
  4. URL/絵文字/コードブロックを除去して言語判定
  5. langdetect 信頼度 ≥ TRANSLATE_MIN_CONFIDENCE のみ続行
  6. en → 日本語、ja → 英語、それ以外はスキップ
  7. チャンネルmodeチェック（both/en2ja/ja2en/off）
  8. プロバイダで翻訳 → reply
  9. 失敗時は ⚠️ リアクションのみ（チャンネル静音）

⚙送料計算ロジック

計算手順

  1. 商品リストから商品合計重量を算出: Σ(weight_g × qty)
  2. 梱包材 PACKAGING_WEIGHT_G（既定1000g）を加算
  3. kg換算後、0.5kg刻みで 切り上げ: math.ceil(total_kg / 0.5) * 0.5
  4. 20kg超の場合: 自動で複数箱に分割発送（各箱 商品19kg + 梱包1kg = 20kg）
  5. スプシから国 × 重量帯の 安い方 列を取得 → DHL or FedEx 確定
  6. 料金を返す（分割時は箱ごとの料金を合算）

計算例

例1: 単一発送 
ポケモンBOX × 5個 = 1,500g
+ Single card × 10枚 = 30g
+ 梱包材 = 1,000g
= 総重量 2,530g → 3.0kg帯
→ DHL ¥9,400 (アメリカ宛)
例2: 分割発送 
ポケモンCASE × 6個 = 24,000g
→ 2箱に分割 (各12,000g + 梱包1kg = 13,000g)
→ 各箱 13.0kg帯で計算 → 合計料金

スプシ仕様

  • シート: 比較表US基準
  • B列 = 重量(kg)、C列以降 = 国ごと3列ブロック (DHL/FedEx/安い方)
  • 0.5kg刻み、最大68kg対応
  • 5分メモリキャッシュ。/shippingadmin reload で即時更新

🌍複数サーバー運用

サーバーごとに 独立したプロセス として動作。各 deployments/<server>/ に別の Discord token / API キー / チャンネル ID を設定。

追加サーバーのセットアップ

cp -r deployments/template deployments/myserver
cp deployments/myserver/.env.example deployments/myserver/.env
# .env を編集して値を埋める
# 必要なら credentials/service_account.json を配置

./run.sh deployments/myserver

並列実行

# systemd / Docker / PM2 などで永続化
./run.sh deployments/server-tokyo &
./run.sh deployments/server-osaka &
./run.sh deployments/server-test &

詳細は SETUP.md を参照。
