Discord サーバークローン — セットアップガイド

support_bot 機能（翻訳・意図判定・送料計算・チケット・Welcome・Giveaway）込みで、サーバー構造を別サーバーへ複製する手順

  4. ③ .env に値を書き込む（記入場所一覧）
  5. ④ 実行手順
  6. ⑤ クローン後：support_bot を新サーバーで動かす
  7. ⑥ うまくいかない時

① 全体像 — どのトークン・APIが、どこに必要か

クローン作業と、クローン先で support_bot を動かすために必要なものを一枚にまとめます。

何 どこで取る 記入場所（ファイル / 変数名） 用途
必須Discord Bot Token discord.com/developers → Bot タブ support_bot/scripts/.env の DISCORD_TOKEN_CLONE= クローン実行
必須Source / Target Guild ID Discord アプリ：開発者モードON → サーバー名右クリック → IDをコピー 同 .env の SOURCE_GUILD_ID= / TARGET_GUILD_ID= どこからどこへ複製するか
必須support_bot 用 Bot Token 同上（同じ bot を使い回しても、別 bot を作っても可） support_bot/deployments/<新サーバー名>/.env の DISCORD_TOKEN_SUPPORT= 新サーバーで bot を動かす
機能別DeepL API Key deepl.com/pro-api → アカウント → Auth Key deployments/<新サーバー名>/.env の DEEPL_API_KEY= 翻訳（DeepL 使用時）
機能別Anthropic API Key console.anthropic.com → API Keys 同 .env の ANTHROPIC_API_KEY= 意図判定 / 翻訳フォールバック
機能別Google サービスアカウント JSON console.cloud.google.com → IAM → サービスアカウント → キー作成（JSON） JSON を deployments/<新サーバー名>/credentials/service_account.json に配置。.env の GOOGLE_SERVICE_ACCOUNT_JSON= でパス指定 送料計算（Sheets 読込）
機能別Google Sheet ID シートURL /d/ここ/edit の部分 同 .env の SHIPPING_SHEET_ID= 送料計算
ポイント：クローン用 bot と support_bot 用 bot は、同じ bot を兼用してもOKです。同じ bot を source / target 両方に入れて、クローン後はそのまま target で稼働させればトークン1本で済みます。

② Discord Bot を作る（クローン用）

  1. Discord Developer Portal を開く。
  2. 右上 New Application → 名前を入れて作成（例：Server Clone Bot）。
  3. 左メニュー Bot → Reset Token → 表示されたトークンをコピー。
    必須このトークンを DISCORD_TOKEN_CLONE に入れます。
  4. Privileged Gateway Intents：構造コピーのみなら全部OFFで動作します（オン要件なし）。
  5. 左メニュー OAuth2 → URL Generator：
    • SCOPES: bot ✅
    • BOT PERMISSIONS: Administrator ✅（最も簡単）
      最小権限で運用したい場合は Manage Channels + Manage Roles + Manage Server + View Channels
    画面下に出る URL をコピー。

③ 両方のサーバーに bot を入れる

  1. 手順②で生成した URL を、まず コピー元サーバーで開く → サーバー選択 → 認証。
  2. 同じ URL をもう一度開き、今度は コピー先サーバーを選んで認証。
  3. コピー先サーバーで、bot のロールを サーバー設定 → ロール から 一番上に移動させる。
    必須これをしないと、bot より上位の @everyone やロールのパーミッションを操作できず、複製が部分的に失敗します。
  4. Discord アプリで 設定 → 詳細設定 → 開発者モード をON。サーバー名を右クリック → サーバーIDをコピー。
    source / target 両方のIDをメモ。
注意：コピー先サーバーは原則「空の新サーバー」を推奨。既存サーバーに重ねるなら --wipe-target オプション（次章）で既存チャンネル/ロールを全削除してから複製する形になります。戻せません。

④ .env に値を書き込む（記入場所一覧）

テンプレを .env としてコピーして埋めます：

cd ~/Desktop/ClaudeCode/Discordbot/support_bot
cp scripts/.env.clone.example scripts/.env

scripts/.env の中身：

# 必須3項目
DISCORD_TOKEN_CLONE=MTAxxx...（手順②でコピーした bot トークン）
SOURCE_GUILD_ID=123456789012345678
TARGET_GUILD_ID=987654321098765432

# オプション
CLONE_ROLES=1   # 1=ロールも複製 / 0=target 既存ロールに名前マッチさせる
CLONE_ICON=0    # 1=アイコン・バナーも複製 / 0=しない
探索順：scripts/.env → support_bot/.env → 親ディレクトリ .env の順に読み込まれます。既存の bot 用 .env に直接書いてもOK。

⑤ 実行手順

  1. 初回は必ず dry-run（何が作られるかだけ表示、変更なし）： 
    cd ~/Desktop/ClaudeCode/Discordbot/support_bot
python scripts/clone_server.py --dry-run
    出力例： 
    === Clone Plan ===
source: My Source Server (123...)
target: My New Server (987...)
dry_run=True wipe=False

[1/3] Cloning roles ...
  [dry] @everyone perms <- 1071698660929
  [dry] create role: Admin (color=#5865F2, hoist=True)
  [dry] create role: VIP (color=#EB459E, hoist=True)
  ...
[2/3] Cloning categories and channels ...
  [dry] category: ENTRANCE (overrides=0)
  [dry] TextChannel: rule-guide (under: ENTRANCE)
  ...
  2. 内容OKなら本実行： 
    python scripts/clone_server.py
  3. 既存の target を一度キレイにしてから複製したい場合： 
    python scripts/clone_server.py --wipe-target
    確認実行時に yes 入力が必要。target の既存チャンネル/ロールが全削除されます。
クローンされるもの：
  • ロール（色／hoist／mentionable／パーミッション、@everyone はパーミッションのみ上書き）
  • カテゴリ
  • テキスト・ボイス・フォーラム・ステージ チャンネル（topic, slowmode, NSFW, bitrate, user_limit）
  • チャンネル単位の権限オーバーライド（ロール基準）
  • 任意：サーバーアイコン・バナー（CLONE_ICON=1）
クローンされないもの（Discord API の制約）：
  • 過去のメッセージ／メンバー／メンバーのロール割り当て
  • Webhook、連携アプリ、スレッド本文
  • managed ロール（bot 自身のロール等は target 側で自動付与）
  • メンバー個別の権限オーバーライド（target にそのユーザーがいる保証がないため）

⑥ クローン後：support_bot を新サーバーで動かす

構造をコピーしただけでは bot は新サーバーで何もしません。新サーバー専用の deployment を作って各IDを書き直す必要があります。

  1. deployment ディレクトリを作成： 
    cd ~/Desktop/ClaudeCode/Discordbot/support_bot
cp -r deployments/template deployments/<新サーバー名>
  2. 新サーバーで開発者モードON → 必要なチャンネル/カテゴリ/ロールを右クリックでIDコピー → deployments/<新サーバー名>/.env に貼る。
  3. 記入が必要なID（最低限）：
    変数何のID
    GUILD_ID新サーバー本体
    MODERATOR_CHANNEL_ID#moderator-only
    COMMUNITY_CHANNEL_ID#free-talk（意図判定の監視先）
    PRODUCT_INQUIRY_CHANNEL_ID#ticket
    SHIPPING_GUIDE_CHANNEL_ID#shipping-guide
    TRANSLATE_CHANNEL_IDS自動翻訳対象（free-talk, questions, card-game, delivery-room, welcome 等カンマ区切り）
    TICKET_CATEGORY_IDSCreated / Claimed / Closed Tickets カテゴリ（カンマ区切り）
    TICKET_CATEGORY_ID新規チケット作成先カテゴリ
    TICKET_STAFF_ROLE_IDS運営ロール（カンマ区切り）
    WELCOME_CHANNEL_ID#welcome
    WELCOME_AUTOROLE_IDS入室時に自動付与するロール
    CARD_GAME_CHANNEL_ID#card-game
  4. API キー類を埋める：DISCORD_TOKEN_SUPPORT / DEEPL_API_KEY / ANTHROPIC_API_KEY / SHIPPING_SHEET_ID。
  5. Google サービスアカウント JSON を deployments/<新サーバー名>/credentials/service_account.json に配置。シートの「共有」でこのサービスアカウントのメールアドレスに閲覧権限を付与。
  6. bot 起動： 
    DEPLOYMENT=<新サーバー名> python main.py
    （または run.sh 内の deployment 名を書き換え）
memo：deployments/test/.env を見れば、テスト環境で実際に何が埋まっているか参考になります。コピー＆置換するのが早いです。

⑦ うまくいかない時

症状原因 / 対処
ERROR: bot が source/target サーバーに参加していません 手順③の招待がどちらか抜けている。両方のIDを再確認。
! forbidden creating role: XXX bot のロールが target の対象ロールより下にある。bot ロールをロール一覧の一番上に。
! @everyone perms update forbidden bot に Manage Server 権限が無い。招待時に Administrator か該当権限を付与。
カテゴリは出来たがチャンネルが入っていない カテゴリ作成と並行でAPIがレートリミット中。再実行 or 少数ずつ流すと安定。
クローン後、bot がスラッシュコマンドを認識しない support_bot を起動 → 数分待つ。Discord 側のコマンド同期に時間がかかる。main.py 初回起動時に同期される設計。
送料計算が「シートが読めません」 サービスアカウントのメールにシート閲覧権限が付与されていない。シートを開いて「共有」→ アドレス追加 →「閲覧者」。
